# Contribution guide

Follow this workflow to update standards.

## Process

1. Identify gap or inconsistency.
2. Draft proposal MDX in a feature branch.
3. Reference existing principles; avoid conflicting guidance.
4. Add examples (Do / Don't) with rationale.
5. Request review from design + content + engineering.
6. On approval, update index navigation ordering.

## Criteria for acceptance

- Aligns with voice and tone
- Reduces ambiguity
- Improves consistency or accessibility
- Localizable without phrase breaks

## Versioning

Add a dated entry in `changelog.mdx` for each change.
